# 声明式UI思想入门

> 如果你写过 React/Vue，Compose 的声明式 UI 几乎不需要学。
> 如果你只写过 Java 后端，可以把 Compose 理解为「把 Thymeleaf 模板写进了 Java 代码里，数据变了模板自动重新渲染」。

---

## 1. 声明式 vs 命令式

**命令式（传统 Android View）**：你告诉系统「怎么做」。

```java
// 命令式 — 一步步操作 UI 控件
TextView textView = findViewById(R.id.textView);
textView.setText("Hello");         // 手动更新
Button button = findViewById(R.id.button);
button.setOnClickListener(v -> {   // 手动绑定事件
    textView.setText("Clicked");
});
```

**声明式（Compose）**：你描述 UI「长什么样」，框架负责更新。

```kotlin
// 声明式 — 描述 UI 的样子，数据变化时自动更新
@Composable
fun Greeting() {
    var text by remember { mutableStateOf("Hello") }
    Column {
        Text(text)                    // 你只描述：这里有一个 Text，内容 = text
        Button(onClick = { text = "Clicked" }) {  // 你只描述：点击按钮时更新 text
            Text("Click me")
        }
    }
    // text 变化 → Text 自动更新（框架负责）
}
```

**核心公式**：**UI = f(state)** — 你是定义 f（UI 长什么样），框架负责在 state 变化时重新调用 f。

**后端类比**：声明式 UI ≈ SQL。你写 `SELECT * FROM users WHERE age > 18`，数据库决定怎么执行，不会告诉它用哪个索引、怎么扫描。

## 2. Compose 函数的特点

一个普通的 `@Composable` 函数：

```kotlin
@Composable
fun NewsItem(news: News, onClick: () -> Unit) {
    // 1. 用 Column 垂直排列子元素
    Column(modifier = Modifier.padding(16.dp)) {
        // 2. 标题
        Text(
            text = news.title,
            style = MaterialTheme.typography.titleMedium
        )
        // 3. 摘要
        Text(
            text = news.summary,
            style = MaterialTheme.typography.bodyMedium,
            maxLines = 2,
            overflow = TextOverflow.Ellipsis
        )
        // 4. 分隔线
        Divider()
    }
}
```

**Composable 函数的特征**：

| 特征 | 说明 |
|------|------|
| 必须有 `@Composable` 注解 | 告诉编译器这是 Composable |
| 可被「重组」 | 状态变化时框架会重新调用它 |
| 可重入 | 同一个函数可能被调用多次（类似递归的暂停和恢复） |
| 无返回值 | 返回 `Unit`，通过 emit 把 UI 发送到屏幕上 |
| 调用顺序不确定 | 框架可能按任意顺序、并发调用（但 UI 结果是确定的） |

**重要规则**：Composable 函数中**不能做耗时操作**（网络请求、文件读写）。因为这些操作会在重组时重复执行。

## 3. Compose vs 传统 Android View 体系

| 对比维度 | 传统 View (XML + findViewById) | Compose |
|---------|-------------------------------|---------|
| UI 定义方式 | XML 定义 + 代码绑定 | 纯 Kotlin 代码，无 XML |
| 更新方式 | 手动 findViewById + setText | 数据变化自动重组 |
| 状态管理 | 手动同步 | 框架自动追踪 |
| 列表 | RecyclerView + Adapter + ViewHolder | LazyColumn（更简单） |
| 自定义 View | 继承 View / ViewGroup | @Composable 函数组合 |
| 学习曲线 | 需要学 XML、View 体系、Adapter 模式 | 需要学声明式思维 |

**Compose 不是 View 的包装**：Compose 渲染到 Canvas 上，不依赖原生 View 树。这也是为什么 Compose 性能更好（减少了 View→XML→inflate 的链路）。

## 4. 基础布局组件

```kotlin
// Column：垂直布局（类比线性布局 LinearLayout vertical）
Column {
    Text("第一行")
    Text("第二行")
    Text("第三行")
}

// Row：水平布局（类比 LinearLayout horizontal）
Row {
    Icon(imageVector = Icons.Default.Star, contentDescription = null)
    Text("收藏")
}

// Box：层叠（类比 FrameLayout / CSS position:absolute）
Box {
    Image(painter = painterResource(R.drawable.background), contentDescription = null)
    Text("浮在图片上方的文字", modifier = Modifier.align(Alignment.Center))
}

// Spacer：空白占位
Spacer(modifier = Modifier.height(16.dp))

// Modifier：修饰符链（类比 CSS 的各种属性）
Text("Hello",
    modifier = Modifier
        .padding(16.dp)           // 外边距
        .fillMaxWidth()           // 填充宽度
        .clickable { onClick() }  // 可点击
)
```

## 5. Compose 的「无 XML」哲学

在传统 Android 中：`XML 布局 → findViewById → 代码绑定 → setText/listener`

在 Compose 中：`@Composable 函数 = 布局 + 数据 + 事件`，三合一。

这带来的好处：
- 不需要在 XML 和 Kotlin 之间切换
- 不需要写 `findViewById`（Null 安全由 Kotlin 直接保证）
- 不需要 DataBinding（Compose 自带状态绑定）
- 编译时检查（用错了组件/属性编译就报错，而不是运行时）

**后端类比**：传统 XML+Code ≈ JSP（模板和逻辑分离），Compose ≈ JSX（模板和逻辑在同一处）。

## 6. 第一个 Composable 完整示例

```kotlin
@Composable
fun SimpleCounter() {
    // Step 1: 创建可以被 Compose 追踪的状态
    var count by remember { mutableStateOf(0) }

    // Step 2: 描述 UI
    Column(
        modifier = Modifier
            .padding(16.dp)
            .fillMaxWidth(),
        horizontalAlignment = Alignment.CenterHorizontally
    ) {
        Text(
            text = "Count: $count",
            style = MaterialTheme.typography.headlineMedium
        )
        Spacer(modifier = Modifier.height(16.dp))
        Button(onClick = { count++ }) {
            Text("+1")
        }
    }
    // count 变化 → Column 块内的所有 Text 自动更新
}
```

**数据流**：用户点击按钮 → `count` 变化 → Compose 检测到 State 变化 → 重新执行 `SimpleCounter()` → Text 显示新值。
