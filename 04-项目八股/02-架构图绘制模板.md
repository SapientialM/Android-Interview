# 架构图绘制模板

> 答辩 PPT 中最重要的两页：架构分层图 + 数据流图。
> 推荐工具：draw.io（免费）、Excalidraw（手绘风）、PlantUML（代码生成）。

---

## 1. 总体架构图（三层分层）

```
┌──────────────────────────────────────────────────┐
│ UI Layer (Compose Screens)                       │
│ ┌──────────┐ ┌──────────┐ ┌──────────┐         │
│ │ Home     │ │ Detail   │ │ Settings │         │
│ │ Screen   │ │ Screen   │ │ Screen   │         │
│ └────┬─────┘ └────┬─────┘ └────┬─────┘         │
│      │ 订阅 StateFlow  │           │              │
├──────┼──────────┼───────────┼──────────────────┤
│ ViewModel Layer                                 │
│ ┌────┴─────┐ ┌────┴─────┐ ┌────┴─────┐         │
│ │ HomeVM   │ │ DetailVM │ │ Settings │         │
│ │ StateFlow│ │ StateFlow│ │  VM      │         │
│ └────┬─────┘ └────┬─────┘ └──────────┘         │
│      │ 调用 Repository    │                      │
├──────┼──────────┼──────────────────────────────┤
│ Data Layer                                      │
│ ┌────┴──────────────┐                           │
│ │   Repository      │  ← 数据聚合、缓存策略     │
│ └──┬─────────────┬──┘                           │
│    │             │                               │
│ ┌──┴──────┐ ┌───┴───────┐                       │
│ │Retrofit │ │ Room(可选)│                       │
│ │Network  │ │Local DB   │                       │
│ └─────────┘ └───────────┘                       │
├──────────────────────────────────────────────────┤
│ DI Layer (Hilt)                                 │
│ ┌──────────────────────────────────────┐        │
│ │ NetworkModule / DatabaseModule / ...  │        │
│ └──────────────────────────────────────┘        │
└──────────────────────────────────────────────────┘
```

**PPT 中怎么画**：用 3 个水平块 + 箭头。每个块内放 2-3 个关键词（如 UI 层放 "Compose Screen" "StateFlow 订阅"，ViewModel 层放 "StateFlow" "事件处理"）。

## 2. 数据流图（单向数据流）

```
用户点击按钮
       │
       ▼ (事件向上)
┌─────────────┐
│  ViewModel  │
│  fun onXXX(){}
└──────┬──────┘
       │ 调用
       ▼
┌─────────────┐    suspend fun    ┌──────────────┐
│ Repository  │ ────────────────→ │  Retrofit    │
│             │ ←──────────────── │  (Network)   │
└──────┬──────┘   返回数据         └──────────────┘
       │
       │ StateFlow.value = newState
       ▼
┌─────────────┐    collectAsState     ┌────────────┐
│ StateFlow   │ ──────────────────→  │  Compose   │
│ (ViewModel) │      (状态向下)        │  Screen    │
└─────────────┘                       └─────┬──────┘
                                            │ 重组
                                            ▼
                                        UI 更新
```

**PPT 中怎么画**：用箭头 + 方框，从左到右或从上到下都可以。关键标注「事件向上，状态向下」。

## 3. 模块依赖图（如果用模块化）

```
app (主模块)
 ├── Hilt DI
 ├── Navigation
 └── 依赖 ↓

:core:network      :core:data        :core:ui
 (Retrofit+OkHttp)  (Repository 接口)  (Loading/Error/Empty 组件)

:feature:news-list  :feature:news-detail  :feature:settings
 (新闻列表)          (新闻详情)            (设置页面)
```

**注意**：如果项目没有用模块化，可以画成「包结构图」代替。

## 5. 画图的 3 个经验

1. **不要放太多细节**：架构图展示的是结构，不是把所有类名列出来
2. **颜色区分层次**：UI 层一种色，ViewModel 层一种色，Data 层一种色
3. **箭头不要太密**：只画关键数据流（1-2 条），其他的口头补充即可
