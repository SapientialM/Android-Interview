# 00-15天速成规划

> 为 Java 后端转 Android 客户端量身定制的 17 天冲刺排期。
> 核心原则：**面试考什么学什么、用后端知识类比、借助 AI Coding 提效**。

---

## 背景

你是一个有 Java 后端经验的工程师，零 Android 基础，需要在约 17 天内完成：
1. 一个类今日头条的新闻客户端项目（Kotlin + Compose + MVVM）
2. 掌握面试高频八股（Handler、Binder、Activity、Compose、协程、内存泄漏等）
3. 准备项目答辩、拿到终面机会

## 总体策略

| 优先级 | 事项 | 目标 | 时间分配 |
|:---:|------|------|:---:|
| P0 | 客户端项目 | 功能完整、架构规范、答辩出彩 | 60% |
| P1 | 基础速成 | 掌握面试常考的 Android/Kotlin 八股 | 25% |
| P2 | 其他项目 | 保底交付即可 | 15% |

**关键原则**：项目开发借助 AI Coding，你把控架构设计和技术选型；节省出的时间全部投入基础补齐 + 答辩预演。

## 如何使用本规划

1. **Day 1 开始前**：先阅读 [README.md](../README.md) 中的「Java 后端 → Android 概念映射表」建立全局认知
2. **每天学习时**：打开对应阶段文档，按「白天项目 + 下午基础 + 晚上复习」推进
3. **学习资源**：
   - 优先阅读本仓库的整合文档（[01-Kotlin-速成/](../01-Kotlin-速成/)、[02-Compose-UI体系/](../02-Compose-UI体系/) 等）
   - 需要深入了解时，按 [06-资源索引/](../06-资源索引/) 查找原始资料
4. **检验标准**：每天睡前用文档中的「今日自测」自问自答，答不上来第二天优先复习

## 后端类比学习策略

Android 中几乎每个概念都能在后端找到对应。本规划的核心学习方法是：

> 遇到新概念 → 先问「这在后端里是什么？」→ 建立映射 → 理解差异 → 内化

详细映射表见 [README.md 概念映射表](../README.md#java-后端--android-概念映射表)。
