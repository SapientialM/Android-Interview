## Day 15：模拟面试

### 上午：自问自答（2h）

对着 [04-项目八股/04-常见追问FAQ](../04-项目八股/04-常见追问FAQ.md) 逐题回答。

**必须准备的 5 个核心问题，写出逐字稿**：

1. **为什么选 Compose 而不是传统 XML？**
   > 声明式 UI 更契合 Kotlin 语言特性，状态驱动 UI 减少样板代码。对比传统方式需要在代码和 XML 之间做映射（findViewById/DataBinding），Compose 一个 @Composable 函数就是 UI。另外，它是 Google 推荐的未来方向，新项目应该选它。

2. **AI Coding 帮你做了什么？你自己做了什么？**
   > AI 帮我生成模板代码（Retrofit 接口定义、基础 UI 布局），节省了重复劳动。但架构设计（MVVM 分层、状态管理方案）、技术选型（Paging3 分页、Coil 缓存）、性能优化和边界情况处理都是我自己独立完成的。

3. **列表有 10 万条数据，怎么保证不卡顿？**
   > 三个层面：分页加载（Paging3 只加载可见页）、ViewHolder 复用（LazyColumn 自动复用 + key 确保高效 Diff）、图片管控（Coil size 限定按需加载、内存缓存+磁盘缓存、滑动时暂停加载）。

4. **你的项目数据流怎么走？**
   > 单向数据流：UI 层通过 StateFlow 订阅 ViewModel 的状态，用户事件通过 ViewModel 处理，ViewModel 调用 Repository 获取数据（网络/DB），Repository 返回结果后更新 StateFlow，UI 自动重组。

5. **这个项目让你学到了什么？**
   > 从零学 Kotlin 和 Compose，理解了声明式 UI 的范式（对比后端 MVC）。最大收获是架构设计：MVVM + 单向数据流让代码清晰可测试，依赖注入（Hilt）让模块解耦。性能优化方面，理解了列表复用、图片缓存、重组优化这些移动端特有的挑战。
