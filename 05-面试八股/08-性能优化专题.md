# 性能优化专题

> 级别：⭐⭐ 高频。面试中「你们做了哪些性能优化」是开放式高频题。

---

## 面试官怎么问

- 「你们 App 做了哪些性能优化？启动时间优化了多少？」
- 「卡顿怎么定位和优化？」
- 「包体积优化有哪些手段？」

---

## 1. 启动优化 ⭐⭐⭐

### 冷启动流程

```
点击图标 → fork 进程 → Application.onCreate() → Activity.onCreate()
          → createWindow → setContentView → 首帧(首帧)
```

### 优化手段

| 阶段 | 优化 | 效果 |
|------|------|------|
| Application | 延迟初始化（非必需组件异步加载） | 减少 onCreate 耗时 |
| Application | 线程池初始化、避免 I/O 阻塞主线程 | 减少等待 |
| Activity | Splash 预加载主题（windowBackground 设品牌色） | 视觉秒开 |
| Activity | 布局优化（避免嵌套、merge/include/ViewStub） | 减少 inflate |
| Activity | 首帧后再加载首页数据（LaunchedEffect） | 不阻塞首帧 |

### 启动时间检测

```bash
adb shell am start -W com.example.app/.MainActivity
# WaitTime: 总耗时
# ThisTime: 最后一个 Activity 启动耗时
# TotalTime: 所有 Activity 启动耗时
```

## 2. UI 渲染优化

| 手段 | 说明 |
|------|------|
| 减少布局层级 | 传统 View：用 ConstraintLayout 代替多层嵌套。Compose：天然扁平 |
| 减少 Overdraw | 用 GPU 过度绘制检测工具（开发者选项），去掉不必要的背景 |
| ViewStub 延迟加载 | 不立即显示的 View 延迟 inflate |
| 避免 Measure/Layout 过多 | 传统 View：onMeasure/onLayout 耗时可用 systrace 分析 |

## 3. 卡顿优化

### 卡顿原因

- 主线程做 IO / 复杂计算
- 布局层级过深
- 动画频繁触发 Measure/Layout
- GC 频繁（内存抖动）
- 过度绘制

### 卡顿检测

| 工具 | 说明 |
|------|------|
| systrace / Perfetto | 系统级 trace，看各线程在做什么 |
| BlockCanary | 基于 Looper.loop() 的 Printer 检测主线程阻塞 |
| Choreographer | 检测丢帧（帧率 < 60 或 < 16.67ms/帧） |
| Android Profiler | IDE 内置，看 CPU/内存/网络实时状态 |

### Compose 特有优化

- 加 `key` 避免列表错位重组
- `@Stable` / `@Immutable` 减少无效重组
- `derivedStateOf` 避免重算
- `remember` 缓存 Modifier

## 5. 包体积优化

| 手段 | 效果 |
|------|------|
| 代码混淆（ProGuard/R8） | 减少 5-10% |
| 资源压缩（shrinkResources） | 减少 3-5% |
| 图片转 WebP | 减少 0.5-2 MB |
| 动态下发 SO | 按 CPU 架构分包（App Bundle） |
| 删除无用资源 | Lint 检查 + 手动清理 |

---

## 6. 网络优化

| 手段 | 说明 |
|------|------|
| 连接池复用 | OkHttp 默认复用 |
| DNS 优化（HTTPDNS） | 防劫持、减少 DNS 解析延迟 |
| 压缩 | Gzip / Brotli |
| 接口合并 | 减少请求次数 |
| 缓存 | OkHttp 自带缓存 + 业务缓存 |

---

## 7. Battery（耗电优化）

| 手段 | 说明 |
|------|------|
| 减少网络唤醒 | WorkManager 代替定时任务 |
| 减少 GPS 使用 | 用 `requestLocationUpdates` 的 minTime 参数 |
| 后台限制 | Android 8.0+ 后台执行限制 |
| JobScheduler | 系统调度延迟任务，合并唤醒时间 |

## 后端类比

| Android 性能优化 | 后端 |
|-----------------|------|
| 启动优化 | 服务启动时间优化（Spring Boot warm-up） |
| UI 卡顿 | API 响应时间 > 1s → 超时 |
| 内存泄漏 | 连接池、线程、缓存未释放 |
| 包体积 | Docker 镜像大小（多阶段构建） |
| 网络优化 | CDN、数据压缩、连接池 |
| 耗电优化 | 服务器功耗优化 |

## 延伸追问

1. **启动优化的量化指标？**
   → 目标：冷启动 < 2s（用户可接受），优秀 < 1s。用 Systrace 精确测量每个阶段耗时。

2. **怎么检测卡顿？**
   → Choreographer 丢帧检测。Printer 机制检测 Message 处理时长。systrace 看完整系统调用链。

3. **你项目中做了哪些性能优化？**
   → 准备 2-3 个具体例子（列表滑动流畅度、图片缓存策略、启动时间优化），说清楚「优化前 → 手段 → 效果」。
