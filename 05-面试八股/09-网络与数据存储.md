## 1. OkHttp 拦截器链

```
请求 → 应用拦截器 (Request)
       → 网络拦截器 (Request)
         → OkHttp 核心（DNS、连接池、发送请求）
       → 网络拦截器 (Response)
     → 应用拦截器 (Response)
→ 响应
```

| 拦截器类型 | 用途 |
|-----------|------|
| 应用拦截器 (addInterceptor) | 日志、统一 Header、重试 |
| 网络拦截器 (addNetworkInterceptor) | 缓存处理、桥接（header 补充） |

**后端类比**：拦截器链 ≈ Spring Security Filter Chain / Servlet Filter 链。每层只做自己的事，链式调用。

## 2. Retrofit 动态代理原理

和 OpenFeign 完全一致：

1. `retrofit.create(NewsApi::class.java)` → JDK 动态代理
2. 调用 `newsApi.getNews()` → Proxy.invoke()
3. 解析注解（@GET、@Query 等）→ 构建 OkHttp Request
4. 调用 OkHttp 发送请求
5. 拿到 Response → ConverterFactory 反序列化 JSON → 返回

## 3. SharedPreferences → DataStore

### SharedPreferences 的缺陷

- 在 UI 线程进行 I/O（可能 ANR）
- 没有类型安全（全存储为 String/Set）
- commit() 同步写磁盘 → 卡顿

### DataStore（推荐替代）

```kotlin
// DataStore 是协程原生的、异步的、类型安全的 K-V 存储
val Context.dataStore by preferencesDataStore("settings")

class SettingsRepository @Inject constructor(
    private val dataStore: DataStore<Preferences>
) {
    val theme: Flow<Theme> = dataStore.data.map { prefs ->
        Theme.valueOf(prefs[THEME_KEY] ?: "LIGHT")
    }

    suspend fun setTheme(theme: Theme) {
        dataStore.edit { prefs ->
            prefs[THEME_KEY] = theme.name
        }
    }
}
```

| 对比 | SharedPreferences | DataStore |
|------|------------------|-----------|
| 线程 | 主线程 I/O（可能 ANR） | 协程异步 |
| 类型安全 | String/Set 手动转换 | Preferences.Key<Integer> 等 |
| 迁移 | — | 自动迁移 SharedPreferences |

## 4. Room（SQLite ORM）

```kotlin
@Entity(tableName = "news")
data class NewsEntity(
    @PrimaryKey val id: String,
    val title: String,
    val content: String,
    val timestamp: Long
)

@Dao
interface NewsDao {
    @Query("SELECT * FROM news ORDER BY timestamp DESC")
    fun getAllNews(): Flow<List<NewsEntity>>  // 返回 Flow = 自动刷新

    @Insert(onConflict = OnConflictStrategy.REPLACE)
    suspend fun insertAll(news: List<NewsEntity>)
}

@Database(entities = [NewsEntity::class], version = 1)
abstract class AppDatabase : RoomDatabase() {
    abstract fun newsDao(): NewsDao
}
```

**后端类比**：Room ≈ JPA/Hibernate（编译期 SQL 校验），`@Entity` ≈ `@Table`，`@Dao` ≈ Spring Data JPA Repository。

## 延伸追问

1. **OkHttp 连接池怎么复用？**
   → OkHttp 默认维护一个连接池，通过相同的 Address（host + port + scheme）复用连接。空闲连接保持 5 分钟。

2. **DataStore 和 Room 分别用在什么场景？**
   → DataStore 存小量 K-V（设置项、Token）。Room 存结构化数据（新闻列表、离线缓存）。

3. **Room 的 Flow 自动刷新原理？**
   → Room 在底层对数据库表做了 trigger，数据变化时自动通知 Flow 的收集者。
